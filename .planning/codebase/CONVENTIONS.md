# Coding Conventions - MiniFootballLeagueAnalyzer

## Languages
- **Python**: PEP 8 standards, `snake_case` for variables and functions, `PascalCase` for classes.
- **JavaScript/TypeScript**: Standard ESM modules, `camelCase` for variables, `PascalCase` for React components.

## Documentation
- **Bilingual**: Main documentation in Spanish (`README.md`) and English (`README_EN.md`).
- **Comments**: Extensive descriptive comments in Spanish within the Python scripts to explain algorithmic decisions (e.g., ELO multipliers).

## Styling & UI
- **Vanilla CSS**: Avoidance of utility frameworks like Tailwind for maximum control.
- **Aesthetics**: High-contrast dark mode, glassmorphism, and fluid animations using Framer Motion.
- **SEO**: Semantic HTML5, descriptive meta tags, and optimized heading hierarchy.

## Git Workflow
- **Atomic Commits**: Small, focused commits with descriptive messages.
- **Conventional Commits**: Suggested for clarity (e.g., `feat:`, `fix:`, `docs:`).
